## Детальное описание — проблемы #1–#10

### #1 — Описание содержит чужеродные детали

**Файл:** `scripts/generate-noscut.ts`

Промпт для DeepSeek просит написать описание ноуската и перечисляет структуру
(состав → совместимость → срок → опт), но не передаёт фактический состав комплекта.
DeepSeek берёт из собственных знаний то, что считает комплектом для конкретной модели
и ошибается: для Prado J150 сгенерировало текст про втулки стабилизатора и тяги
(ходовая часть), для Lexus RX AL20 — про тормозные диски и датчики износа (тормозная
система). Константа COMPONENTS определена в скрипте, но в промпт не передаётся.

**Решение.** Явно вписать состав в промпт:

Напиши описание ноуската для ${m.make} ${m.model} ${m.generation} (${m.yearStart}–${m.yearEnd}).
Ноускат — это комплект для восстановления передней части автомобиля. Состав комплекта
строго фиксирован: бампер, оптика (фары и противотуманки), радиатор, телевизор
(рамка радиатора), датчики, камера. Никаких других деталей — только эти шесть позиций.
Напиши 80–120 слов на русском без заголовков. Структура:
1. Состав комплекта (перечисли именно эти 6 деталей в тексте)
2. Совместимость с модификациями ${m.model} ${m.generation}
3. Срок поставки 30 дней под заказ
4. Условия для оптовых покупателей

**Важно о порядке генерации.** На момент написания ТЗ в каталоге сгенерировано только
2 модели. После исправления промпта эти две записи перегенерируются первыми (с флагом
--force и --limit=2 — см. #3). Все последующие модели генерируются уже с исправленным
промптом. Отдельного шага «полная перегенерация всего каталога» не требуется.

### #2 — Промпт рыночной цены ищет не тот продукт

**Файл:** `scripts/update-noscut-prices.ts`

Текущий промпт просит найти «среднюю рыночную цену ноуската». Слово «ноускат»
в российском интернете ассоциируется с б/у деталями с авторазбора — DeepSeek находит
именно такие объявления (100–180 тысяч рублей). Эти цифры не отражают реальную
альтернативу для покупателя и создают ситуацию, когда наша цена выглядит выше рыночной.

Реальная альтернатива для покупателя — собрать аналогичный комплект самостоятельно,
купив каждую из 6 деталей по отдельности. Именно суммарную стоимость такой покупки
нужно находить и показывать как рыночный ориентир.

**Решение.** Новый промпт для fetchPrice():

Найди суммарную стоимость покупки запасных частей для восстановления передней части
${entry.make} ${entry.model} ${entry.generation} на российском рынке.
Нас интересует суммарная цена этих 6 деталей, купленных по отдельности:
1. Передний бампер в сборе
2. Фары (комплект: левая + правая)
3. Радиатор охлаждения
4. Телевизор (рамка передней части / front panel)
5. Парктроники / датчики парковки
6. Камера переднего вида
Источники: exist.ru, autodoc.ru, Авито, oem.ru.
Верни ТОЛЬКО JSON: { "price": number | null, "sources": string[] }
где price — суммарная медианная цена в рублях за все 6 позиций
(null если меньше 2 источников дали данные хотя бы по 4 из 6 позиций),
sources — список источников (1–3 строки). Без markdown, только JSON.

### #3 — Изображения: мусорные подписи на выносках

**Файл:** `scripts/generate-noscut.ts`

Промпт просит numbered callouts и labeled components. Qwen-Image-2.0-Pro не умеет
надёжно генерировать читаемый текст внутри изображения — это системное ограничение
диффузионных моделей. Результат: «Fest», «L.lcat», «ALBCUT», «Frame Frame:» —
мусор, подрывающий доверие к карточке.

**Решение.** Убрать все инструкции про текст, подписи, цифры и выноски. Новый промпт:

Technical exploded-view illustration of ${m.make} ${m.model} ${m.generation} front end noscut kit.
Light gray neutral background. Show the vehicle silhouette in 3/4 front angle.
The 6 noscut components are visually separated and floating slightly away from the car body:
front bumper, headlights pair, radiator, front panel frame, parking sensors, front camera.
All other parts of the car body are faded/transparent.
Clean technical product illustration style. No text, no numbers, no labels, no callouts whatsoever.
High quality, sharp edges, professional catalog look.

Для принудительной перегенерации существующих 2 записей добавить в скрипт флаг --force,
который отключает resume-логику (не пропускает slug даже если jpg уже существует).

### #4 — Город Владивосток в блоке доставки

**Файл:** `src/lib/deliveryConfig.ts`

Компания расположена в Приморском крае (рядом с Владивостоком). Включение своего города
в список доставки нелогично для клиента: самая дешёвая и быстрая доставка — в точку
нашего присутствия.

**Решение.** Заменить Владивосток на Краснодар — крупнейший город юга России (~1,1 млн),
максимально географически удалённый от нас. Новые данные:
{ name: "Краснодар", priceFrom: 13000, daysMin: 22, daysMax: 32 }

Файл помечен @rule: Do NOT edit manually. Данное изменение вносится точечной правкой
напрямую (не регенерацией) — замена одного города не требует полного прогона скрипта.
После правки добавить комментарий:
// @important: Vladivostok excluded — company location. Replaced with Krasnodar.

### #8 — Текст блока оптовиков: неверные обещания, узкая аудитория

Заголовок: «Для оптовых покупателей»

из Азии — не только то, что есть в каталоге. Цена зависит от объёма и обсуждается персонально.

### #10 — Рыночная цена отображается когда она ниже нашей

**Файлы:** `src/app/catalog/noscut/[slug]/page.tsx`, `src/components/noscut/NoscutCard.tsx`

Строка «Рынок: ~X ₽» отображается при любом ненулевом marketPriceRu независимо от
соотношения с нашей ценой. Если рыночная цена ниже 199 000 ₽ — страница демонстрирует,
что мы дороже рынка. Бейдж «Экономия X%» уже защищён условием marketPriceRu > priceFrom,
но сама строка с ценой не имеет такой же проверки.

**Решение.** Добавить условие в оба файла:
{entry.marketPriceRu && entry.marketPriceRu > entry.priceFrom && (
  <p>Рынок: ~{entry.marketPriceRu.toLocaleString("ru-RU")} ₽ {...}</p>
)}

### #11 — Плашка «Автомобили» на хабе /catalog не кликабельна

**Файл:** `src/app/catalog/page.tsx`

На хабе /catalog плашка «Ноускаты» — это Link, плашка «Автомобили» — просто div.
Клик по ней ничего не делает. Обе плашки визуально выглядят одинаково — пользователь
ожидает, что обе кликабельны. Это нарушает принцип консистентного поведения UI.

**Решение.** Обернуть плашку «Автомобили» в Link href="#catalog" — прокрутка к секции
с сеткой автомобилей. Добавить id="catalog" к секции с CatalogClient если его ещё нет.
Простой Link, не требует превращения страницы в client component.

### #12 — Подвал: навигация растянута в одну длинную колонку

**Файл:** `src/components/layout/Footer.tsx`

Footer рендерит NAV_ITEMS в одну вертикальную колонку. Два пункта с дочерними

### #13 — Ноускаты полностью отсутствуют в sitemap.xml

**Файл:** `src/app/sitemap.ts`

В sitemap нет ни /catalog/noscut, ни /catalog/noscut/[slug]. Все 131 страница ноускатов
вне поля зрения поисковых роботов — они могут индексироваться только через краулинг
ссылок, что медленнее и ненадёжнее.

**Решение.** Добавить два блока в sitemap.ts:

Статичный:
{ url: "https://jckauto.ru/catalog/noscut", lastModified: new Date(),
  changeFrequency: "daily", priority: 0.8 }

Динамический — читать /var/www/jckauto/storage/noscut/noscut-catalog.json через
fs.readFileSync и генерировать:
noscutEntries.map((entry) => ({
  url: `https://jckauto.ru/catalog/noscut/${entry.slug}`,
  lastModified: new Date(entry.updatedAt),
  changeFrequency: "weekly" as const,
  priority: 0.7,
}))

При пустом или отсутствующем файле — возвращать пустой массив, не бросать ошибку.

### #15 — Эндпоинт /api/lead без rate limiting

**Файл:** `src/app/api/lead/route.ts`

Эндпоинт принимает POST-запросы без ограничений по частоте. rateLimiter.ts существует
и подключён к AI-инструментам, но не к форме лидов. Любой скрипт может флудить
Telegram-группу фейковыми заявками.

**Решение.** Подключить checkRateLimit и recordUsage из src/lib/rateLimiter.ts.
IP получать из x-forwarded-for (первый в списке) или x-real-ip, фолбэк — "unknown".
Лимит: 5 заявок с одного IP в 24 часа.

Ключ передавать с префиксом lead:${ip} (не просто ${ip}) — чтобы не конкурировать
с AI-лимитами, которые используют тот же usageMap без префикса.

При превышении: статус 429, тело:
{ "error": "Слишком много запросов. Попробуйте позже или позвоните нам напрямую: ${CONTACTS.phone}" }
Телефон брать из CONTACTS.phone, не хардкодить.

### #16 — Нет валидации формата телефона на бэкенде

**Файл:** `src/app/api/lead/route.ts`

На бэкенде только if (!phone) — любая непустая строка проходит. Можно отправить "x"
или произвольный текст — он улетит в Telegram как заявка.

**Решение:**
const cleanPhone = phone.replace(/[^\d\s\+\-\(\)]/g, "");
const digits = cleanPhone.replace(/\D/g, "");
if (digits.length < 10) {
  return NextResponse.json({ error: "Некорректный номер телефона" }, { status: 400 });
}
В Telegram отправлять cleanPhone, не исходный phone.

### #17 — Нет ограничения длины текстовых полей на бэкенде

**Файл:** `src/app/api/lead/route.ts`

Поля name, message, subject принимают строку любого размера. Поле message в тысячи
символов нежелательно в Telegram-сообщении.

**Решение.** Обрезать с пометкой [truncated]:
- name: максимум 100 символов
- message: максимум 1000 символов
- subject: максимум 200 символов

Обрезка (.slice(0, limit) + "[truncated]") предпочтительнее отказа запроса —
пользователь не получает ошибку, в Telegram уходит усечённая версия.

### #18 — Поле телефона в форме без маски ввода

**Файл:** `src/components/LeadForm.tsx`

Placeholder +7 (___) ___-__-__ обещает маску, которой нет. Пользователи вводят номера
в разных форматах без обратной связи о правильности. Для конверсионной формы это
потеря доверия и снижение конверсии.

**Решение.** Подключить react-input-mask (или imask + react-imask при проблемах с
peer dependencies). Маска: +7 (999) 999-99-99. Совместимость с react-hook-form через
inputRef. Выбор библиотеки на усмотрение исполнителя — обязательна проверка npm install
без ошибок peer dependencies.

### #19 — Нет кнопки сброса фильтров на /catalog/noscut

**Файл:** `src/app/catalog/noscut/page.tsx`

При 0 результатах по фильтрам нет способа сбросить все три фильтра одним действием.
На мобильном сброс вручную — это 6 тапов вместо одного.

**Решение.** Кнопка «Сбросить», видимая только при activeFilterCount > 0:
{activeFilterCount > 0 && (
  <Link
    href="/catalog/noscut"
    className="flex items-center gap-2 rounded-lg border border-border px-4 py-2 text-sm font-medium text-text-muted transition-colors hover:bg-gray-50"
  >
    Сбросить
  </Link>
)}

### #20 — Задержка анимации карточек накапливается

**Файл:** `src/components/noscut/NoscutCard.tsx`

delay: index * 0.1 — при 24 карточках последняя появляется через 2.3 с. Эффект
каскада нужен, но такая задержка воспринимается как зависание страницы.

**Решение.**
transition={{ delay: Math.min(index, 4) * 0.08 }}
Карточки 0–4: задержка 0–0.32 с. Карточки 5+: появляются вместе с четвёртой.
Максимальная задержка 0.32 с, эффект каскада сохраняется.
